# Proximal Policy Optimization(PPO)

OpenAI默认的强化学习算法, 是Policy Gradient的变形。

# On-Policy到Off-Policy

在RL中我们要学习的就是一个Agent

  • On-Policy: 和环境互动的Agent就是我们要学习的Agent； 一边做互动，一边学习。
  • Off-Policy: 和环境互动的Agent不是我们要学习的Agent； 通过看别人学习。

前面提到的Policy Gradient的方法就是一种On-Policy的方法。

∇R‾θ=Eτ∼pθ(τ)[R(τ)∇logpθ(τ)]\nabla \overline{R}_{\theta}=E_{\tau \sim p_{\theta}(\tau)}\left[R(\tau) \nabla \log p_{\theta}(\tau)\right] ∇​R​​​​θ​​=E​τ∼p​θ​​(τ)​​[R(τ)∇logp​θ​​(τ)]

前面提到了Policy Gradient的方法要玩很多次游戏才更新依次参数θ\thetaθ,这样子就会变得超级不划算，有没有更合理的方法呢?

  • Goal: Using the sample from πθ′\pi_{\theta'}π​θ​′​​​​ to train θ\thetaθ. θ′\theta'θ​′​​ is fixed, so we can re-use the sample data.

# 重要性抽样 Importance Sampling

我们已经知道:

Ex∼p[f(x)]≈1N∑i=1Nf(xi)E_{x \sim p}[f(x)] \approx \frac{1}{N} \sum_{i=1}^{N} f\left(x^{i}\right) E​x∼p​​[f(x)]≈​N​​1​​​i=1​∑​N​​f(x​i​​)

其中, xix^ix​i​​ 是从p(x)p(x)p(x)采样得到。但是假设现在的问题是，我们没有办法从p(x)p(x)p(x)里面采样数据, 我们只能从q(x)q(x)q(x)里面采样，那怎么算这个期望呢?

Ex∼p[f(x)]=∫f(x)p(x)dx=∫f(x)p(x)q(x)q(x)dx=Ex∼q[f(x)p(x)q(x)]E_{x \sim p}[f(x)] = \int f(x) p(x) d x=\int f(x) \frac{p(x)}{q(x)} q(x) d x=E_{x\sim q}\left[f(x) \frac{p(x)}{q(x)}\right] E​x∼p​​[f(x)]=∫f(x)p(x)dx=∫f(x)​q(x)​​p(x)​​q(x)dx=E​x∼q​​[f(x)​q(x)​​p(x)​​]

这样就是从qqq这个分布里面采样得到xxx, 而不是从ppp里面做采样了，真是神奇的变化啊! 这里的p(x)q(x)\frac{ p(x)}{q(x)}​q(x)​​p(x)​​就是起到一个修正的作用的权值。

理论上分布q(x)q(x)q(x)为什么都可以,但是实作上qqq和ppp不能差太多。虽然两个分布的期望是一样的，但是他们的方差呢?

=Ex∼p[f(x)2p(x)q(x)]−(Ex∼p[f(x)])2=E_{x \sim p}[f(x)^{2} \frac{p(x)}{q(x)}]-\left(E_{x \sim p}[f(x)]\right)^{2} =E​x∼p​​[f(x)​2​​​q(x)​​p(x)​​]−(E​x∼p​​[f(x)])​2​​

两式中只有第一项不一样。因此，若是两个分布太大，就会出现很大的偏差。若是采样不够多就有可能出现偏差, 举个例子来说:

本来两者的期望按理来说应该是一样的，但是由于两个概率分布p(x)p(x)p(x), q(x)q(x)q(x) 相差太大，若是采样不够就会出现得到的两个分布的的期望 不一样。

# 正式开始将On-Policy变成Off-Policy

疑问

可以这样理解吗? 原本τ\tauτ服从的分布pθ(τ)p_{\theta}(\tau)p​θ​​(τ)其实是不清楚的, 需要玩很多很多把的游戏才能够统计近似出来。而且每改变一次 θ\thetaθ, 又要玩很多把游戏才能得到新的pθ(τ)p_{\theta}(\tau)p​θ​​(τ), 这样做是非常耗费时间的。于是我们就使用一个已知的概率分布去抽样，得到的 游戏记录可以保存起来，这样就可以不断地拿去更新。

将原来的:

∇R‾θ=Eτ∼pθ(τ)[R(τ)∇logpθ(τ)]\nabla \overline{R}_{\theta}=E_{\tau \sim p_{\theta}(\tau)}\left[R(\tau) \nabla \log p_{\theta}(\tau)\right] ∇​R​​​​θ​​=E​τ∼p​θ​​(τ)​​[R(τ)∇logp​θ​​(τ)]

变成:

∇R‾θ=Eτ∼pθ′(τ)[pθ(τ)pθ′(τ)R(τ)∇logpθ(τ)]\nabla \overline{R}_{\theta}=E_{\tau \sim p_{\theta^{\prime}}(\tau)}\left[\frac{p_{\theta}(\tau)}{p_{\theta^{\prime}}(\tau)} R(\tau) \nabla \log p_{\theta}(\tau)\right] ∇​R​​​​θ​​=E​τ∼p​θ​′​​​​(τ)​​[​p​θ​′​​​​(τ)​​p​θ​​(τ)​​R(τ)∇logp​θ​​(τ)]

这里的θ′\theta'θ​′​​是来给θ\thetaθ做示范的, 实际当中都是拿θ′\theta'θ​′​​来和环境做互动。

  • 使用θ′\theta'θ​′​​采样得到数据
  • 使用采样得到的数据去训练θ\thetaθ很多次

从Policy Gradient的方法中，我们知道

∇R‾θ≈E(st,at)∼πθ[Aθ(st,at)∇logpθ(atn∣stn)]\nabla \overline{R}_{\theta} \approx E_{\left(s_{t}, a_{t}\right) \sim \pi_{\theta}}\left[A^{\theta}\left(s_{t}, a_{t}\right) \nabla \log p_{\theta}\left(a_{t}^{n} | s_{t}^{n}\right)\right] ∇​R​​​​θ​​≈E​(s​t​​,a​t​​)∼π​θ​​​​[A​θ​​(s​t​​,a​t​​)∇logp​θ​​(a​t​n​​∣s​t​n​​)]

利用重要性采样的技术，我们可以得到:

=E(st,at)∼πθ′[pθ(at∣st)pθ′(at∣st)pθ(st)pθ′(st)Aθ′(st,at)∇logpθ(atn∣stn)]=E_{\left(s_{t}, a_{t}\right) \sim \pi_{\theta^{\prime}}}\left[\frac{p_{\theta}\left(a_{t} | s_{t}\right)}{p_{\theta^{\prime}}\left(a_{t} | s_{t}\right)} \frac{p_{\theta}\left(s_{t}\right)}{p_{\theta^{\prime}}\left(s_{t}\right)} A^{\theta^{\prime}}\left(s_{t}, a_{t}\right) \nabla \log p_{\theta}\left(a_{t}^{n} | s_{t}^{n}\right)\right] =E​(s​t​​,a​t​​)∼π​θ​′​​​​​​[​p​θ​′​​​​(a​t​​∣s​t​​)​​p​θ​​(a​t​​∣s​t​​)​​​p​θ​′​​​​(s​t​​)​​p​θ​​(s​t​​)​​A​θ​′​​​​(s​t​​,a​t​​)∇logp​θ​​(a​t​n​​∣s​t​n​​)]

这里我们认为pθ(st)pθ′(st)=1\frac{p_{\theta}(s_t)}{p_{\theta'}(s_t)}=1​p​θ​′​​​​(s​t​​)​​p​θ​​(s​t​​)​​=1,这样就可以约掉，这里可以这样强行解释一下:

# Add Constrain

使用下面的技巧，在做训练的时候添加一个Constrain, 我们不直接训练Jθ′(θ)J^{\theta^{\prime}}(\theta)J​θ​′​​​​(θ),而是训练下面的目标函数:

JPPOθ′(θ)=Jθ′(θ)−βKL(θ,θ′)J_{P P O}^{\theta^{\prime}}(\theta)=J^{\theta^{\prime}}(\theta)-\beta KL\left(\theta, \theta^{\prime}\right) J​PPO​θ​′​​​​(θ)=J​θ​′​​​​(θ)−βKL(θ,θ​′​​)

这里的βKL(θ,θ′)\beta KL\left(\theta, \theta^{\prime}\right)βKL(θ,θ​′​​)是两个模型的输出的动作的的KL Diversion，他是用来衡量θ\thetaθ和θ′\theta^\primeθ​′​​像不像的量, 这个值是一个非负数，当θ\thetaθ和θ′\theta^\primeθ​′​​相等时, KL(θ,θ′)=0KL\left(\theta, \theta^{\prime}\right)=0KL(θ,θ​′​​)=0, 相差越大这个值越大。

注意

这里的KL Diversion并不是参数上的距离，而是行为上的距离。因为有很多时候，参数变化了不一定行为就发生了改变。
